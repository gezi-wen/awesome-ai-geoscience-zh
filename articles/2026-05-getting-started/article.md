# AI + 地学技术写作：从入门到变现

> 大二学生如何用技术写作积累作品集、建立行业影响力、拿到实习机会。真实路线，不带滤镜。

---

## 为什么是 AI + 地学？

三个理由让这个交叉领域成为技术写作的蓝海：

1. **赛道窄、竞争少**——会遥感的不一定写得好，写得好的不一定懂 AI。两样都占的人极少
2. **项目全是开源的**——TorchGeo、DeepEarth、EVE、Prithvi……顶级的工具和模型都在 GitHub 上，没有付费墙
3. **正好在做考研准备**——你的地学知识一边写一边巩固，写出来的文章可以直接放进考研复试的作品集

---

## 可以写什么？

不要觉得非要原创研究才能写。技术写作的门类很广：

| 类型 | 难度 | 例子 |
|------|------|------|
| 入门教程 | ⭐ | 「TorchGeo 10 分钟上手」 |
| 架构解析 | ⭐⭐ | 「DeepEarth 时空编码器是怎么工作的」 |
| 源码解读 | ⭐⭐⭐ | 「Earth4D.py 逐行解读」 |
| 论文翻译 | ⭐⭐ | 「Prithvi 论文要点中译」 |
| 工具评测 | ⭐ | 「5 个遥感数据集加载库横评」 |
| 实战案例 | ⭐⭐⭐ | 「用 DeepEarth 预测野火风险」 |

**建议从「入门教程」和「架构解析」开始**——这两个类型不需要 GPU、不需要跑通完整实验，只要你能读懂代码和论文就能写。

## 发在哪里？

### 中文平台

| 平台 | 优势 | 劣势 |
|------|------|------|
| **掘金** | AI/技术标签精准、编辑推荐机制 | 纯开发者社区 |
| **知乎** | 覆盖面广、能被学术界看到 | 算法不可控 |
| **CSDN** | 搜索引擎收录好 | 界面体验差 |
| **个人博客** | 完全控制 | 从零建流量 |

**建议**：掘金 + 知乎双发。同一篇文章先在掘金发，隔两天改一改摘要发知乎。

### 英文平台

| 平台 | 优势 | 要求 |
|------|------|------|
| **freeCodeCamp** | 百万级受众 | 英文、质量高 |
| **Medium** | 写作者社区 | 英文、会员制 |
| **dev.to** | 开发者友好 | 英文 |

英文写作是通往 Outreachy 和 remote 机会的钥匙。不一定每篇都发英文，但至少准备 1-2 篇高质量英文文章作为「门面」。

## 怎么找到可写的项目？

日常监控这几个渠道：

1. **GitHub Trending** — 每周看一下 `topic:geospatial` `topic:remote-sensing` 的新项目
2. **arXiv** — 搜 `cs.CV + remote sensing` / `cs.LG + geospatial`，新论文 → 新解读素材
3. **HuggingFace** — `ibm-nasa-geospatial`、`eve-esa` 等组织的新模型
4. **Write the Docs Slack** — 全球技术写作者社区，有工作机会频道
5. **OSGeo Discourse** — GRASS GIS、QGIS 等开源 GIS 项目的社区

## 从写作到变现

技术写作的变现路径不是立竿见影的，但路径是清晰的：

```
第一阶段（1-3 个月）
  写 5-10 篇中文教程 → 建立掘金/知乎影响力
  收获: 关注者、作品集

第二阶段（3-6 个月）
  写 1-2 篇英文教程 → 投稿 freeCodeCamp / Medium
  加入 Write the Docs 社区
  收获: 国际曝光、行业人脉

第三阶段（6-12 个月）
  申请 Outreachy / GSoC 等有报酬的实习
  接 freelance 技术写作项目
  收获: $7000+ 实习津贴 / $500-2000 单篇稿费
```

**每一篇文章都是一张名片**。你写的 TorchGeo 教程被某位研究员 Google 到——这可能就是一次连接。
